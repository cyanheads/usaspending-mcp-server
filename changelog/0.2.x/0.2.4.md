# 0.2.4 — 2026-07-16

Three `usaspending_disaster_spending` bug fixes, plus adoption of the `@cyanheads/mcp-ts-core` ^0.10.14 toolchain (Socket supply-chain scanning, pinned Docker base, re-synced skills).

## Fixed

- **`usaspending_disaster_spending` geography returned empty rows (#18)** — the geography dimension mapped a phantom `aggregated_amount` field the API never sends and omitted the `spending_type` the endpoint requires (returning HTTP 422). It now sends `spending_type: obligation` with `scope: place_of_performance` and reads the real `amount` field, surfacing obligated dollars plus `award_count`, `population`, and `per_capita` per area, rendered as columns in the results table.
- **`usaspending_disaster_spending` breakdown pagination was ignored (#19)** — the agency, CFDA, and recipient breakdowns passed `limit`/`page` at the top level of the request body, which USAspending silently drops, always returning the first default page. Pagination is now nested under a `pagination` key so `page` and `limit` take effect.
- **`usaspending_disaster_spending` unreachable `no_data` error contract (#35)** — removed the `no_data` error entry the handler never throws, and rewrote the empty-results guidance, which told callers to remove the `def_codes` filter that every non-overview dimension actually requires.

## Changed

- **Geography `spending_type` output is now `obligation`** — the geography dimension reports the measure it queries; the `award`/`total` input still applies to the agency and recipient dimensions. Tool and field descriptions now spell out per-dimension applicability of `spending_type` and `limit`.
- **Biome cleanup from the 0.10.14 toolchain** — dropped redundant `async` on `USASpendingService` passthrough methods and replaced non-null assertions with explicit guards; `usaspending_get_agency` now throws a typed `missing_input` when neither `toptier_code` nor `agency_slug` resolves. Behavior-preserving.
- **Framework 0.10.14 toolchain adopted** — `bunfig.toml` adds a 3-day `minimumReleaseAge` install guard (framework excluded for same-day adoption) and the `@socketsecurity/bun-security-scanner`; `Dockerfile` pins the `oven/bun:1.3.14` base and adds BuildKit cache mounts; devcheck scripts, vendored skills, the changelog template, and repo metadata (`LICENSE`, `.github/SECURITY.md`, `.gitattributes`) re-synced.
