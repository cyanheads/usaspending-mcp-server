# 0.2.6 — 2026-07-16

Four fixes across recipient search and autocomplete: a stale IDV-awards continuation signal, missing recipient-search pagination, a broken recipient-autocomplete field mapping, and a limit ceiling too low for the endpoint it wraps.

## Added

- **`usaspending_search_recipients` accepts a `page` input** — output gains `page_metadata` (`total`, `page`, `has_next`, `limit`), replacing the old cap-guessing truncation heuristic; recovery guidance at the max limit now points to the next page instead of suggesting an impossible limit increase. ([#24](https://github.com/cyanheads/usaspending-mcp-server/issues/24))
- **`usaspending_autocomplete`'s `recipient` type surfaces `uei` and `duns`** as independent optional output fields, each populated only when the search text matches that specific identifier. ([#25](https://github.com/cyanheads/usaspending-mcp-server/issues/25))

## Changed

- **`usaspending_autocomplete`'s `limit` ceiling raised from 50 to 500** — matches the upstream `recipient` lookup's documented max; the other four lookup types (`naics`, `psc`, `cfda`, `awarding_agency`) have no documented ceiling and return only their matching entries regardless. ([#30](https://github.com/cyanheads/usaspending-mcp-server/issues/30))
- **`RawRecipientSearchResult`'s dead `location`/`state_province` fields removed** — confirmed against the live `POST /recipient/` endpoint to never populate. ([#24](https://github.com/cyanheads/usaspending-mcp-server/issues/24))

## Fixed

- **`usaspending_get_idv_awards` no longer hides later pages behind a stale upstream `hasNext`** — `page_metadata.has_next`, `enrichment.has_next_page`, and the truncation notice now also fire whenever a page is full (`results.length >= limit`), not only when upstream `hasNext` is true; a short or empty page still marks the end. ([#23](https://github.com/cyanheads/usaspending-mcp-server/issues/23))
- **`usaspending_autocomplete`'s `recipient` type no longer renders matches as empty objects** — the handler read nonexistent `recipient_id`/`legal_business_name` fields; it now maps the live response's `recipient_name` to `name`. ([#25](https://github.com/cyanheads/usaspending-mcp-server/issues/25))
