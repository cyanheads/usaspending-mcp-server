# 0.1.5 — 2026-05-30

## Changed

- **`usaspending_search_awards`** — `output.message` removed; pagination totals (`total`, `page`, `has_next`) and empty-result recovery now surface via `ctx.enrich()` into the typed `enrichment` block, reaching both `structuredContent` and `content[]`. `no_match` error contract removed — empty results return success with an enrichment notice ([#10](https://github.com/cyanheads/usaspending-mcp-server/issues/10)).
- **`usaspending_search_recipients`** — `output.message` removed; recipient count and empty-result notice now via `enrichment`. `no_match` error contract removed — empty results return success with an enrichment notice ([#10](https://github.com/cyanheads/usaspending-mcp-server/issues/10)).
- **`usaspending_spending_by_category`** — `output.message` removed; pagination totals and empty-result notice now via `enrichment`. `no_data` error contract removed — empty results return success with an enrichment notice ([#10](https://github.com/cyanheads/usaspending-mcp-server/issues/10)).
- **`usaspending_disaster_spending`, `usaspending_get_award_subawards`, `usaspending_get_award_transactions`, `usaspending_list_agencies`, `usaspending_spending_by_geography`, `usaspending_spending_over_time`, `usaspending_autocomplete`** — enrichment block added; result counts, pagination metadata, and contextual guidance reach both channels.
- **`@cyanheads/mcp-ts-core`** `^0.9.13` → `^0.9.16` — picks up `ctx.enrich()` / `ctx.enrich.notice()` API.
