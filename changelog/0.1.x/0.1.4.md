# 0.1.4 — 2026-05-28

## Changed

- **`usaspending_get_agency` missing-input error** — `missing_input` reason now maps to `ValidationError` instead of `InvalidParams` (`JsonRpcErrorCode.ValidationError`). Clients reading `data.reason` see the same `'missing_input'` string; clients reading the numeric error code see the updated value. Test updated to match.
- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.9.13` — picks up:
  - `MCP_HTTP_MAX_BODY_BYTES` — configurable inbound request-body cap (default 1 MiB); oversized requests rejected with HTTP 413 before the SDK parses the body ([#157](https://github.com/cyanheads/mcp-ts-core/issues/157))
  - HTTP session-init gate — stateful HTTP mode rejects non-`initialize` requests without `Mcp-Session-Id` with HTTP 400 ([#154](https://github.com/cyanheads/mcp-ts-core/issues/154))
  - Expected client errors (401, 403, 400, 404) now log at `warning` level instead of full `ErrorHandler` pipeline with stack traces, reducing noise ([#158](https://github.com/cyanheads/mcp-ts-core/issues/158))
  - `GET /mcp` surfaces `package.json` `keywords` on the status JSON alongside name/version/description ([#165](https://github.com/cyanheads/mcp-ts-core/issues/165))
  - `landing.requireAuth` — landing page inventory now gated behind auth by default when `MCP_AUTH_MODE` is `jwt`/`oauth`; `landing: { requireAuth: false }` added to `createApp()` call to preserve public-catalog behavior ([#156](https://github.com/cyanheads/mcp-ts-core/issues/156))
- **`package.json` `keywords`** — added `typescript`, `bun`, `stdio`, `streamable-http` for better discoverability on npm and `GET /mcp` status.
- **`@biomejs/biome`** `^2.4.15` → `^2.4.16`
