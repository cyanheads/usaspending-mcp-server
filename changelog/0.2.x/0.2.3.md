# 0.2.3 ŌĆö 2026-06-20

Framework-maintenance only ŌĆö no tool, schema, or behavior changes.

## Added

- **`check-dependency-specifiers` devcheck step** ŌĆö new `scripts/check-dependency-specifiers.ts`, registered in `scripts/devcheck.ts` as `Dependency Specifiers` (flag `--no-dep-specifiers`); hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map. Static and local ŌĆö runs in the default and `--fast` passes. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **Plugin marketplace manifest checks in `lint:packaging`** ŌĆö `scripts/lint-packaging.ts` validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` (non-empty descriptions, unscoped machine name in display fields, full package name in the `npx -y` install arg). Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (default on); the `devcheck` Packaging gate now also runs when a plugin manifest is present. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))

## Changed

- **Vendored skills re-synced to framework 0.10.7ŌĆō0.10.9** ŌĆö fourteen `skills/<name>/SKILL.md` files mirrored from upstream, including the `ctx.content` collector reference in `api-context` ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239)) and the `git-wrapup` / `orchestrations` tag-annotation and isolation rewordings.
- **Fresh-scaffold devcheck guards** ŌĆö `scripts/devcheck.ts` skips git-dependent checks (TODOs/FIXMEs, Tracked Secrets, Framework Antipatterns) via an `isGitRepo()` guard when `.git` is absent; `scripts/build-changelog.ts` and `scripts/check-framework-antipatterns.ts` self-guard for direct invocation outside a repo. ([cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [cyanheads/mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243))
- **`scripts/check-skill-versions.ts`** ŌĆö skips a `SKILL.md` deleted from the worktree instead of crashing with ENOENT. ([cyanheads/mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
