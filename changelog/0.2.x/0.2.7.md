# 0.2.7 — 2026-07-16

## Fixed

- **Six tools no longer render the upstream item total as a page count in `content[]`** — `usaspending_disaster_spending`, `usaspending_get_award_subawards`, `usaspending_get_award_transactions`, `usaspending_search_federal_accounts`, `usaspending_search_recipients`, and `usaspending_spending_by_category` previously rendered `**Page:** X of ~Y`, misreading the item total `Y` as a page count; all six now render `**Page:** X | **Total items:** ~Y | **Per page:** Z | **Has next:** Yes/No` through a shared `formatPaginationLine` helper (`src/mcp-server/tools/definitions/pagination.ts`). No `structuredContent` or schema changes — item counts were already correctly typed and described. ([#34](https://github.com/cyanheads/usaspending-mcp-server/issues/34))
