# 0.3.0 ‚Äî 2026-07-17

Three federal-account gaps close together: the award ‚Üí appropriation chain gains its missing link, the program-activity and object-class breakdowns the README advertised now exist behind a tool, and the account response's own component detail is no longer discarded.

## Added

- **`usaspending_get_award_federal_accounts`** ‚Äî lists the Treasury federal accounts that funded an award (`POST /awards/accounts/`), with the amount obligated from each and the funding agency behind it. Each row's `federal_account` (`AGENCY-MAIN`, e.g. `080-0120`) chains into `usaspending_get_federal_account` as `account_code` ‚Äî the award ‚Üí appropriation link `usaspending_get_award`'s `account_obligations_by_defc` (DEF codes, not accounts) left open. Paginated via `page`/`limit` (max 100), with `page_metadata.count` the total across pages. A nonexistent `award_id` returns an empty list with a notice rather than an error: the endpoint answers a miss with HTTP 200 and zero rows, so no not-found contract can fire. ([#28](https://github.com/cyanheads/usaspending-mcp-server/issues/28))
- **`usaspending_get_federal_account_breakdown`** ‚Äî breaks an account's obligations down along a `dimension`: `program_activity` (what the funds support) or `object_class` (what they buy), over `POST /federal_accounts/{code}/program_activities/total` and `.../object_classes/total`. Rows carry `code`, `name`, `obligations`, and ‚Äî for `program_activity` only ‚Äî `type`, either `PAC/PAN` (legacy program activity code/name pair) or `PARK` (Program Activity Reporting Key). Paginated via `page`/`limit` (max 100), with an honest `page_metadata.total`. An account code with no rows on the requested axis returns empty with a notice pointing at the other dimension. ([#31](https://github.com/cyanheads/usaspending-mcp-server/issues/31))

## Fixed

- **`usaspending_get_federal_account` maps `children`** ‚Äî the per-Treasury-Account-Symbol component breakdown was arriving in the response the tool already fetched and being discarded before it reached callers. Each row carries `name`, `code` (the full TAS, e.g. `080-2020/2021-0120-000` ‚Äî the availability-period-scoped component, not an `account_code` this tool accepts), `obligated_amount`, `gross_outlay_amount`, and `budgetary_resources_amount`, rendered in `content[]` at parity with `structuredContent` and omitted entirely when upstream returns none. `RawFederalAccount.children` drops its `unknown[]` placeholder for the real `RawFederalAccountChild[]` row type. ([#41](https://github.com/cyanheads/usaspending-mcp-server/issues/41))
