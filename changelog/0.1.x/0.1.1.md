# 0.1.1 — 2026-05-25

## Fixed

- **`usaspending_autocomplete`** — narrowed `type` enum to valid values (`naics`, `psc`, `cfda`, `recipient`, `awarding_agency`, `funding_agency`); removed overly broad union that accepted arbitrary strings
- **`usaspending_get_federal_account`** — flattened and reduced output schema; removed deeply nested optional fields that upstream rarely populates, fixing sparse-payload failures in field tests
- **`usaspending_disaster_spending`** — relaxed `def_codes` validation to accept any string array; official DEF code list expands with new appropriations and hard-coded enums were rejecting valid newer codes
- **`usaspending_spending_over_time`** — normalized `month` / `calendar_month` fields from upstream response into a single `month` output key; previously missing under certain `group` values
- **`usaspending_search_awards`** — added `award_type_codes` passthrough to support IDV subtypes (`IDV_A`, `IDV_B`, etc.) that the previous union missed
- **`USASpendingService`** — tightened `parseJson` error classification; upstream HTML error pages now surface as `ServiceUnavailable` instead of `SerializationError`
