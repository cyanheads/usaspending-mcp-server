# 0.2.8 — 2026-07-17

Four fixes correcting drift between this server's raw response types and the live USAspending.gov API, across `usaspending_get_award`, `usaspending_get_award_transactions`, `usaspending_get_recipient`, and `usaspending_get_agency`.

## Added

- **`usaspending_get_agency` accepts a `page` input for the sub-agency breakdown** (default 1, 10 per page — the endpoint's natural page cap) — output gains `sub_agency_page_metadata` (`total`, `page`, `has_next`, `limit`) and an `enrichment` block (`sub_agency_page`, `has_more_sub_agencies`, `sub_agency_total`, `notice`) that fires a continuation notice when more pages remain. ([#29](https://github.com/cyanheads/usaspending-mcp-server/issues/29))
- **`usaspending_get_agency` output gains `fiscal_year` and `outlay_amount`** — sourced from the budgetary-resources endpoint's most recent fiscal-year entry. ([#29](https://github.com/cyanheads/usaspending-mcp-server/issues/29))
- **`usaspending_get_recipient` output gains `total_transaction_amount`, `total_transactions`, `total_face_value_loan_amount`, `total_face_value_loan_transactions`, and `location.zip4`** — the current live aggregate and address fields. ([#26](https://github.com/cyanheads/usaspending-mcp-server/issues/26))

## Changed

- **`usaspending_get_agency`'s `budget_authority_amount` renamed to `budgetary_resources_amount`**, now sourced from `GET /agency/{code}/budgetary_resources/` (the most recent fiscal year in `agency_data_by_year[]`) instead of the agency-overview endpoint, which never returns budget data. ([#29](https://github.com/cyanheads/usaspending-mcp-server/issues/29))
- **`usaspending_get_recipient`'s nested `total` object replaced with flat aggregate fields** (see Added) — the live endpoint has never returned a nested `total`. `location.zip5` renamed to `location.zip` to match the live field name. ([#26](https://github.com/cyanheads/usaspending-mcp-server/issues/26))

## Removed

- **`usaspending_get_award`'s and `usaspending_get_agency`'s `transactions_count` output field** — confirmed always absent from both live endpoints; dropped rather than left permanently undefined. ([#12](https://github.com/cyanheads/usaspending-mcp-server/issues/12), [#29](https://github.com/cyanheads/usaspending-mcp-server/issues/29))
- **`usaspending_get_recipient`'s `business_types_description` output field** — the live endpoint only returns `business_types` codes, never descriptions. ([#26](https://github.com/cyanheads/usaspending-mcp-server/issues/26))

## Fixed

- **`usaspending_get_award_transactions`'s empty-result notice cited a nonexistent field** — it told callers to check `transactions_count` from `usaspending_get_award`, a field the live award-detail endpoint never returns; the notice now points callers to verify the `award_id` is a valid `generated_internal_id` from `usaspending_search_awards`. ([#12](https://github.com/cyanheads/usaspending-mcp-server/issues/12))
- **`usaspending_get_agency`'s sub-agency breakdown silently discarded upstream pagination** — the handler sliced the first page to 20 rows and dropped the endpoint's `page_metadata` entirely, leaving callers no way to see or reach the remaining sub-agencies (see Added). ([#29](https://github.com/cyanheads/usaspending-mcp-server/issues/29))
- **`usaspending_get_award` mapped stale raw field names, silently dropping total outlays, base/all-options value, and PSC description** — the live award-detail endpoint returns `total_outlay`, `base_and_all_options`, and `latest_transaction_contract_data.product_or_service_description`, while `RawAwardDetail` expected `total_outlays`, `base_and_all_options_value`, and `product_or_service_code_description`. The tool's public output field names (`total_outlays`, `base_and_all_options_value`, `product_or_service_code.description`) are unchanged — only the internal raw-to-output mapping was broken. ([#36](https://github.com/cyanheads/usaspending-mcp-server/issues/36))
