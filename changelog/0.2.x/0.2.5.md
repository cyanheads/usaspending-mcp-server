# 0.2.5 â€” 2026-07-16

Four `usaspending_search_awards` fixes: a nested `filters` object, the `agency_slug` chaining field, keyset-cursor pagination past the endpoint's 50,000-result window, and a crash fix on the last page of any search.

## Added

- **`usaspending_search_awards` accepts a nested `filters` object** â€” mirrors the analytics-tool shape (`keywords`, `award_type_codes`, `agency_name`, `recipient_name`, `naics_codes`, `time_period_start`/`time_period_end`); per field, a nested value wins over its flat-input equivalent. `recipient_id` is intentionally excluded â€” `spending_by_award` silently ignores it. ([#21](https://github.com/cyanheads/usaspending-mcp-server/issues/21))
- **`agency_slug` in `usaspending_search_awards` results** â€” requested via `AWARD_SEARCH_FIELDS` and mapped into both `structuredContent` and `content[]` for chaining into `usaspending_get_agency`; absent (not `null`) when the agency has no profile page. ([#22](https://github.com/cyanheads/usaspending-mcp-server/issues/22))
- **Keyset-cursor pagination on `usaspending_search_awards`** â€” `last_record_sort_value` + `last_record_unique_id` accepted as input and returned in `page_metadata`, so callers can page past the endpoint's 50,000-result page-number boundary. A new `pagination_limit_exceeded` error fires when `page Ă— limit` crosses that boundary without a cursor supplied. ([#37](https://github.com/cyanheads/usaspending-mcp-server/issues/37))
- **Upstream `messages` surfaced** â€” `usaspending_search_awards` reads the `messages` array `search/spending_by_award/` returns (e.g. an ignored filter) into an `upstream_messages` enrichment field, rendered as a bullet list in `content[]`. ([#21](https://github.com/cyanheads/usaspending-mcp-server/issues/21))
- **Page-based truncation disclosure** â€” `usaspending_search_awards` calls `ctx.enrich.truncated()` with `shown`/`cap` when a page is capped, since the endpoint returns no total count. ([#37](https://github.com/cyanheads/usaspending-mcp-server/issues/37))

## Changed

- **`page_metadata.total` removed from `usaspending_search_awards`** â€” `spending_by_award` never populates it, confirmed across three live query shapes; the `totalCount` enrichment field is removed alongside it. ([#37](https://github.com/cyanheads/usaspending-mcp-server/issues/37))
- **`RawPageMetadata` cursor fields widened to `string | null` / `number | null`** â€” matches the upstream's literal terminal-page sentinels. ([#39](https://github.com/cyanheads/usaspending-mcp-server/issues/39))
- **`USASpendingService.searchAwards()` â€” `page` is now optional**, with new `last_record_sort_value?` / `last_record_unique_id?` params; page and cursor are mutually exclusive on the outgoing request. ([#37](https://github.com/cyanheads/usaspending-mcp-server/issues/37))

## Fixed

- **Crash on the last page of any `usaspending_search_awards` call** â€” the upstream's terminal-page sentinels (`last_record_unique_id: null`, `last_record_sort_value: "None"`) passed a bare `!== undefined` check and failed the `z.number()` output schema, crashing routine searches â€” including zero-result ones â€” once they reached their last page. The cursor pair is now omitted unless `has_next` is true and both values are genuinely usable. ([#39](https://github.com/cyanheads/usaspending-mcp-server/issues/39))
