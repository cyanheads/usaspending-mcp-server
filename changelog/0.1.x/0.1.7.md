# 0.1.7 — 2026-06-04

## Fixed

- **`post()` timeout** ([#13](https://github.com/cyanheads/usaspending-mcp-server/issues/13)) — replaced bare `fetch` with `fetchWithTimeout` in `USASpendingService.post()`, matching the pattern already used in `get()`; `timeoutMs` from server config is now enforced for all POST-based tools (award search, analytics, autocomplete, subawards, transactions, disaster spending).
- **`usaspending_disaster_spending` — `def_codes` validation** ([#7](https://github.com/cyanheads/usaspending-mcp-server/issues/7)) — added a pre-flight guard that throws `ValidationError` when `dimension` is not `"overview"` and `filters.def_codes` is absent or empty; updated schema description to state that `def_codes` is required for `agency`, `cfda`, `recipient`, and `geography` dimensions.
- **`usaspending_spending_by_geography` — empty results** ([#8](https://github.com/cyanheads/usaspending-mcp-server/issues/8)) — converted `no_data` throw to a structured `{ results: [], total: 0 }` return with a `ctx.enrich.notice()` recovery hint; removed the `no_data` error contract entry; behavior is now consistent with `usaspending_search_awards` and `usaspending_spending_by_category`.
- **`usaspending_spending_over_time` — dead-code guard** ([#9](https://github.com/cyanheads/usaspending-mcp-server/issues/9)) — converted unreachable `no_data` throw to a structured return with `ctx.enrich.notice()` for defensive coverage against future upstream behavior changes; removed the `no_data` error contract entry; `ctx.enrich()` call moved before the guard so enrichment is always populated.
