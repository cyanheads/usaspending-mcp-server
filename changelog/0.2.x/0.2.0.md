# 0.2.0 — 2026-06-08

## Added

- **`usaspending_search_federal_accounts`** new tool — keyword and agency-identifier search across federal accounts; returns `account_number` for chaining into `usaspending_get_federal_account` ([#16](https://github.com/cyanheads/usaspending-mcp-server/issues/16))
- **`usaspending_get_idv_awards`** new tool — list child contracts, task/delivery orders, and sub-IDVs placed under an IDV parent award; each row returns `generated_unique_award_id` for chaining into `usaspending_get_award` ([#17](https://github.com/cyanheads/usaspending-mcp-server/issues/17))
- **Applied-filter enrichment echoes** on `usaspending_search_awards`: `applied_keyword`, `applied_agency_name`, `applied_naics_codes`, `applied_time_period_start`, `applied_time_period_end` fields in the enrichment block echo back active filters for agent context ([#11](https://github.com/cyanheads/usaspending-mcp-server/issues/11))
- **Applied-filter enrichment echoes** on `usaspending_spending_by_category`, `usaspending_spending_by_geography`, and `usaspending_spending_over_time`: same `applied_keywords`, `applied_agency_name`, `applied_naics_codes`, `applied_time_period_start/end` fields ([#11](https://github.com/cyanheads/usaspending-mcp-server/issues/11))
- **`notice` enrichment** on `usaspending_get_award_subawards` — fires when the result set is empty, directing callers to check `subaward_count` from `usaspending_get_award` before re-querying ([#12](https://github.com/cyanheads/usaspending-mcp-server/issues/12))
- **`notice` enrichment** on `usaspending_get_award_transactions` — fires when the result set is empty, directing callers to check `transactions_count` from `usaspending_get_award` ([#12](https://github.com/cyanheads/usaspending-mcp-server/issues/12))

## Changed

- **`usaspending_search_awards` `naics_code` renamed to `naics_codes` (array)** — was `z.string()`, now `z.array(z.string())`. Callers passing `naics_code` as a string must update to `naics_codes: ["<code>"]`. Zod strips unknown keys, so the old parameter silently had no effect on upgrade — filter it explicitly or the filter is dropped. ([#14](https://github.com/cyanheads/usaspending-mcp-server/issues/14))
- **`usaspending_get_award` description** updated to cross-reference `usaspending_get_idv_awards` for IDV-category awards.
- **`@cyanheads/mcp-ts-core`** `^0.9.21` held (intentionally pinned; added to `devcheck.config.json` outdated allowlist).

## Fixed

- **`usaspending_disaster_spending` `spending_type` doc** corrected — values are `award` and `total` (not the previously documented `spending` and `loans`). ([#6](https://github.com/cyanheads/usaspending-mcp-server/issues/6))
